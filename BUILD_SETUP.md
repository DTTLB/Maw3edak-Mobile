# Native Build Setup

This document describes the native build configuration for the Maw3edak mobile app.

## Configuration Overview

### Firebase Configuration
- **Android**: `configs/google-services.json`
- **iOS**: `configs/GoogleService-Info.plist`
- Both files are configured in `app.json` and automatically copied during prebuild

### Native Dependencies
All required native modules are configured via Expo config plugins:
- `@react-native-firebase/app` - Firebase core functionality
- `@react-native-firebase/messaging` - Push notifications
- `@react-native-async-storage/async-storage` - Local storage
- `react-native-reanimated` - Animations
- `react-native-screens` - Navigation
- `react-native-safe-area-context` - Safe area handling
- `react-native-gesture-handler` - Gesture support
- `react-native-svg` - SVG rendering

### Android Configuration
- **Package**: `com.maw3edak.app`
- **Google Services**: Configured in `android/build.gradle` (v4.4.1)
- **Min SDK**: As per Expo SDK 54 requirements
- **New Architecture**: Enabled
- **Hermes**: Enabled

### iOS Configuration
- **Bundle ID**: `com.maw3edak.app`
- **Background Modes**: Remote notifications enabled
- **Deployment Target**: iOS 15.1+
- **New Architecture**: Enabled

## Building the App

### Prerequisites
1. Ensure Firebase config files are in the `configs/` directory
2. Run `npm install` to install dependencies
3. Run `npx expo prebuild` to generate native code

### Development Builds
```bash
# Android
npx eas build -p android --profile development

# iOS
npx eas build -p ios --profile development
```

### Preview Builds
```bash
# Android
npx eas build -p android --profile preview

# iOS
npx eas build -p ios --profile preview
```

### Production Builds
```bash
# Android
npx eas build -p android --profile production

# iOS
npx eas build -p ios --profile production
```

## Prebuild

If you need to regenerate native code:
```bash
npx expo prebuild --clean
```

This will:
1. Clean existing `ios/` and `android/` directories
2. Generate fresh native code
3. Copy Firebase config files
4. Configure all native modules

## Troubleshooting

### Firebase config not found
Make sure the config files exist in the `configs/` directory before running prebuild.

### Pod install fails (iOS)
EAS Build handles pod installation automatically. Local iOS builds require macOS.

### Gradle sync fails (Android)
Ensure all environment variables are set in `.env` file.

## EAS Configuration

Build profiles are configured in `eas.json`:
- **development**: Development client with internal distribution
- **preview**: Internal testing builds (APK for Android)
- **production**: Production-ready builds

## Important Notes

1. The `configs/` directory contains sensitive Firebase credentials and should not be committed to version control
2. Native directories (`ios/`, `android/`) are regenerated by `expo prebuild` and can be gitignored
3. All native dependencies are managed via Expo config plugins in `app.json`
4. Push notifications require proper Firebase Cloud Messaging setup
